# Echoes Beneath｜樹狀近身恐怖 RPG

一款以「有狀態故事圖」運作的繁體中文恐怖網頁 RPG。每個節點固定提供 3 個選擇，每一個選擇都會指定不同的下一個節點、所在地、威脅來源與因果狀態；不會再自動回到共同的下一章。

正式網站：<https://echoes-beneath.vercel.app>

## 故事圖規則

- 每個節點固定 3 個選擇，分支因子為 3
- 選擇會建立獨立 `nextNodeKey` 與 `nextContext`
- Context 保存：所在地、上一個所在地、威脅來源、關係、壓力與承接事件
- 移動只能通過場所圖中確實存在的出口
- 留在原地調查時，下一幕仍發生在相同場所
- 故事交織只會在「場所相連、線索相容」時發生
- 程式內建 transition validation，阻止窗邊突然跳到無窗密室等不可能轉場
- 路線數不需要預先存成數百萬筆 JSON；節點由完整路徑與 Context 決定性生成

例如第一個節點的三個選擇會分別進入：

1. 留在書桌前，追查提早行動的倒影
2. 移到房門內側，處理門外沒有影子的來客
3. 前往床頭牆面，追查借用玩家聲音的牆後回聲

這三條路不會自動抵達同一個「第二夜」。

## Cookie 存檔 Schema v3

Browser Cookie 保存：

- 目前節點與故事深度
- 完整路徑紀錄
- 目前所在地與上一個所在地
- 威脅類型、關係與壓力
- 鎮定、警覺、共感
- 線索、物品、隱藏旗標
- 已選擇但尚未解鎖的下一節點
- 重啟次數與最終結局

舊版線性 Cookie 會自動轉換為 v3 根節點；重啟次數仍保留。

## ChatGPT 每小時更新

後續故事由 ChatGPT 排程每小時檢查並更新，不使用 GitHub Actions、GitHub cron 或儲存庫內的自動排程。

每次更新會先讀取目前故事圖、最新故事深度與既有因果狀態，再視內容需要：

- 新增一層連貫的故事後續
- 為新深度加入環境異常、時間扭曲與低語
- 保持每個節點有 3 個不同結果的選擇
- 驗證所在地只能沿真實出口移動
- 只讓場景、線索與威脅相容的路線交織
- 直接提交更新至 GitHub `main`
- 只有程式或靜態資產改變時才重新部署 Vercel

深度 6 理論上已有 `3^6 = 729` 條選擇路徑；後續不需要把 729 筆完整故事全部寫死，故事引擎會根據路徑、所在地與 Context 產生連貫節點。

排程會持續到 2026-07-31 23:00（Asia/Taipei）。最後一次更新會啟用最終狀態，玩家依自己真正走過的場所、威脅與心理路線取得不同版本的結局。

## 本機執行

```bash
npm install
npm run dev
```

手動測試故事圖更新腳本：

```bash
STORY_NOW=2026-07-15T12:00:00+08:00 npm run story:generate
```

此腳本僅供本機或人工維護使用，不由 GitHub Actions 自動執行。

## 圖像資產

網站會優先讀取依「所在地－威脅類型」命名的場景圖片；缺少圖片時使用內建動態 SVG，因此任何分支都不會因沒有對應圖片而空白。

專案另保留 GPT Image 2 資產生成腳本。設定 `OPENAI_API_KEY` 後可執行：

```bash
npm run art:generate
```

## 部署

專案使用 Next.js 並部署於 Vercel。前端會讀取 GitHub 最新的 `public/story.generated.json`；只有程式或靜態資產變更時才需要重新部署。
